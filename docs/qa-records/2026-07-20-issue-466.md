## Visual reference accounting

The implementation was compared against the PRD, the Windows XAML/design-system source, the canonical History, Compact Modes, and About design artifacts, and the current Windows reference captures.

| Dimension | Accounting |
| --- | --- |
| Geometry | Welcome, standard player, chapter panel, Settings (`760×560` logical), About, and compact Mini player geometry followed the reference hierarchy. Mini player geometry was initially suspected, then cleared by a full compositor capture showing the intended ~270 logical-pixel short edge. |
| Spacing | Welcome, History cards, OSC, Settings rail/cards, Media Information, and About used the intended restrained spacing. The Hardware decode switch broke the row rhythm by becoming a tall vertical control. |
| Typography | Headings, labels, monospaced version/build values, and timecodes were consistent and unclipped in the reviewed states. |
| Color/material | Light GTK surfaces, fixed over-video teal, translucent OSC, selection fills, and dark Mini material matched direction. Mini player's video substrate was black instead of live media. |
| Iconography | Core titlebar/OSC/Settings/History icons were coherent. The disabled Scribe placeholder remains visually unfinished but is already owned by #198. |
| Control states | Audio, speed, subtitles, chapter panel, candidate update, private state, and Wayland-unavailable toast behaved. Hardware decode's switch state geometry failed. |
| Behavior | Native choosers, URL, fullscreen action, clipboard, history/private mode, and updates passed. Mini video and EOF teardown failed; physical-pointer-only behavior remains unclaimed. |

## Limitations and holds

- This is a source-only verification. Every `PASS` row above is proven by the
  repository tooling at `cfcab17fe16afc38c03b5f80299ac4acf35dbc75`; no candidate
  bundle was built and no package artifact checksum is bound here.
- The remaining `BLOCKED` rows are the issue's operational-acceptance surface.
  They require the designated self-hosted x86_64 Ubuntu builder — host
  registration, the candidate timer/service, and machine-specific service
  configuration — which is intentionally outside the public repository and
  cannot be produced or inspected by a repository pull request.
- Operational acceptance therefore stays open on the operator: deploy the merged
  tooling to the native builder, confirm the registered timer runs on the
  coalesced cadence, confirm the runtime markers (`candidate-build.json`,
  `last-built.sha`, `last-promoted.sha`) are emitted, and confirm #339 promotes
  the same exact-SHA bundle without a rebuild. This record must not be read as
  clearing those holds.

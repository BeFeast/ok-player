# OK Player Linux `0.1.0-linux-alpha.113`

This is the replacement stable-publish candidate built by Linux Release run
[`29684925884`](https://github.com/BeFeast/ok-player/actions/runs/29684925884). Publication is
blocked by the package-bound failures recorded in issue
[#446](https://github.com/BeFeast/ok-player/issues/446); no release, tag, or feed change is made by
this record.

## User-visible changes

Playback and packaging:

- Intended first stable with the patched libmpv bundled beside both Linux executables. The native
  Wayland DMA-BUF path embeds video into the application surface while retaining the
  `GtkGLArea` fallback for unsupported sessions or initialization failures.
- The compositor-backed 4K60 gate covers HEVC Main10 playback, VA-API, seek and speed recovery,
  fractional-scale geometry, playback clock, and decoder/VO drop growth.
- Candidate-channel watchdog and heartbeat handling distinguish idle, building, stalled, rejected,
  and accepted states without moving the public update feeds.

Desktop behavior and privacy:

- Flatpak installs without usable DRI nodes select the software GTK renderer instead of failing to
  create a render surface.
- History retention supports the shared Forever, 30 days, 90 days, and one year policies. Private
  sessions suppress new history, progress, and poster writes while preserving existing records,
  matching the Windows privacy contract.

## Install, update, rollback, and uninstall

These artifacts are rejected and must not be published or installed as a stable release. After a
fixed build is accepted, the normal Debian and AppImage installation commands remain:

```bash
sha256sum ok-player_0.1.0-linux-alpha.113_amd64.deb
sudo apt install ./ok-player_0.1.0-linux-alpha.113_amd64.deb
```

```bash
sha256sum OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
chmod +x OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
./OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
```

Both lanes use the public Linux update feeds only after publication. Roll back by reinstalling an
older `.deb` or running an older AppImage. Uninstall the Debian package with
`sudo apt remove ok-player`; remove an AppImage by deleting the file. Per-user JSON settings and
history remain available unless the user removes them explicitly.

## Known limitations and blockers

- Both exact package lanes fail the media-backed canonical narrow-width render on the independent
  Ubuntu 26.04 QA execution surface. They log `GTK GLArea error: Failed to create EGL display`;
  the OSC surface mean is `0.0194427` against the required `> 0.055`.
- Both packaged binaries embed source marker `unknown` rather than `192f5e4`, so the installed
  package identity row also fails.
- Native file/folder choosers, drag/drop, clipboard, desktop portals, compositor fullscreen,
  double-click fullscreen, unavailable always-on-top behavior, keyboard focus, and the 4K60
  compositor exercise were not run after the package-bound failures.
- Gapless playback, full HDR handling, and A-B clip/GIF export remain explicitly deferred.
- Other Wayland compositors and GPU/driver combinations may differ from a future accepted GNOME
  environment.

# OK Player Linux `0.1.0-linux-alpha.113`

This stable-publish candidate was built by Linux Release run
[`29705232940`](https://github.com/BeFeast/ok-player/actions/runs/29705232940) from current `main`.
Publication is blocked because both exact stable payloads fail package-bound acceptance. No release,
tag, or feed change is authorized by these notes.

## User-visible changes

Playback and packaging:

- Intended first stable with the patched libmpv bundled beside both Linux executables. The native
  Wayland DMA-BUF path embeds video in the application surface while retaining the `GtkGLArea`
  fallback for unsupported sessions or initialization failures.
- The compositor-backed 4K60 gate covers HEVC Main10 playback, direct VA-API, seek and speed
  recovery, fractional-scale geometry, playback clock, and decoder/VO drop growth.
- Candidate-channel watchdog and heartbeat handling distinguish idle, building, stalled, rejected,
  and accepted states without moving the public update feeds.

Desktop behavior and privacy:

- Flatpak installs without usable DRI nodes select the software GTK renderer instead of failing to
  create a render surface.
- History retention supports Forever, 30 days, 90 days, and one year. Private sessions suppress
  new history, progress, poster, and recent-item writes while preserving existing records, matching
  the Windows retention and privacy contract.

## Install, update, rollback, and uninstall

These artifacts are rejected and must not be published or installed as a stable release. After a
replacement build is accepted, the normal Debian and AppImage installation commands remain:

```bash
sha256sum ok-player_0.1.0-linux-alpha.113_amd64.deb
sudo apt install ./ok-player_0.1.0-linux-alpha.113_amd64.deb
```

```bash
sha256sum OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
chmod +x OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
./OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
```

Both lanes use the public Linux update feeds only after publication. Roll back by reinstalling an
older `.deb` or running an older AppImage. Uninstall the Debian package with
`sudo apt remove ok-player`; remove an AppImage by deleting the file. Per-user JSON settings and
history remain available unless the user removes them explicitly.

## Known limitations and blockers

- Eighteen of nineteen exact-Debian Xvfb rows pass. `xvfb-fullscreen` reaches the required
  `1280x900` geometry and clears fullscreen chrome, but the video frame is black.
- Both exact stable payloads carry the correct version and source marker, and the Debian executable
  resolves its sibling `libmpv.so.2` through `$ORIGIN`. Nevertheless, each live media launch loses
  the Wayland display connection after about six seconds, before selecting
  `native-wayland-dmabuf` or reporting VA-API.
- The accepted candidate `0.11.0-beta.0.73`, built from the same source, survives the control probe,
  selects `native-wayland-dmabuf`, reports VA-API, and emits compositor presentation records.
- Native file/folder choosers, drag/drop, clipboard, desktop portals, compositor fullscreen,
  double-click fullscreen, unavailable always-on-top behavior, keyboard focus, and the 4K60
  compositor exercise are not claimed for the rejected stable bytes.
- Gapless playback, full HDR handling, and A-B clip/GIF export remain explicitly deferred.

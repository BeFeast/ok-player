# OK Player Linux `0.1.0-linux-alpha.113`

This is the stable-publish candidate built by Linux Release run
[`29679405498`](https://github.com/BeFeast/ok-player/actions/runs/29679405498). Publication remains
gated on package-specific live GNOME/Wayland acceptance; no release, tag, or feed change is made by
this record.

## User-visible changes

Playback and packaging:

- First stable-publish candidate with the patched libmpv bundled beside both Linux executables.
  The native Wayland DMA-BUF path embeds video into the application surface while retaining the
  `GtkGLArea` fallback for unsupported sessions or initialization failures.
- The compositor-backed 4K60 gate covers HEVC Main10 playback, VA-API, seek and speed recovery,
  fractional-scale geometry, playback clock, and decoder/VO drop growth.
- Candidate-channel watchdog and heartbeat handling distinguish idle, building, stalled, rejected,
  and accepted states without moving the public update feeds.

Desktop behavior and privacy:

- Flatpak installs without usable DRI nodes select the software GTK renderer instead of failing to
  create a render surface.
- History retention now supports the shared Forever, 30 days, 90 days, and one year policies.
  Private sessions suppress new history, progress, and poster writes while preserving existing
  records, matching the Windows privacy contract.

## Install, update, rollback, and uninstall

Debian/Ubuntu:

```bash
sha256sum ok-player_0.1.0-linux-alpha.113_amd64.deb
sudo apt install ./ok-player_0.1.0-linux-alpha.113_amd64.deb
```

AppImage:

```bash
sha256sum OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
chmod +x OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
./OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
```

Both lanes use the public Linux update feeds after publication. Roll back by reinstalling an older
`.deb` or running an older AppImage. Uninstall the Debian package with `sudo apt remove ok-player`;
remove an AppImage by deleting the file. Per-user JSON settings and history remain available unless
the user removes them explicitly.

## Known limitations

- Native file/folder choosers, drag/drop, clipboard, desktop portals, compositor fullscreen,
  double-click fullscreen, unavailable always-on-top behavior, and keyboard focus require a live
  GNOME/Wayland operator run. Xvfb cannot attest those behaviors.
- The headless AppImage launch used the runtime's extract-and-run fallback because FUSE was not
  available on the VM. Normal FUSE mounting remains part of live-desktop acceptance.
- Gapless playback, full HDR handling, and A-B clip/GIF export remain explicitly deferred.
- Other Wayland compositors and GPU/driver combinations may differ from the accepted GNOME
  environment.

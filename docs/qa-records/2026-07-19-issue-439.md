## Failure evidence

The exact Debian and AppImage payloads independently terminate after approximately six seconds:

```text
Renderer policy: mode=automatic flatpak=false dri-accessible=true backend=automatic hwdec=settings render-api=default gsk-renderer=default
Launch request: 1 item(s), 0 playlist(s), 0 subtitle(s)
Gdk-Message: Error reading events from display: Invalid argument
```

Each stable log contains one initial `native-wayland-egl` session record and four playback samples,
but no compositor presentation, backend-selection, or VA-API record. The accepted same-source
candidate control remains alive for the deliberate 20-second timeout, records
`backend-selected=native-wayland-dmabuf`, `hwdec-current=vaapi`, 793 compositor presents, and zero
decoder-drop growth. The stable Debian runtime manifest contains 171 files versus 161 in the

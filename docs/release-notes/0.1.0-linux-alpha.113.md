[`29947374696`](https://github.com/BeFeast/ok-player/actions/runs/29947374696) from source

## User-visible changes

Playback and packaging:

- Intended first stable with the patched libmpv bundled beside both Linux executables. The native
  Wayland DMA-BUF path embeds video in the application surface while retaining the `GtkGLArea`
  fallback for unsupported sessions or initialization failures.
- The compositor-backed 4K60 gate covers HEVC Main10 playback, direct VA-API, seek and speed
  recovery, Mini-player presentation, playback-clock accuracy, and decoder/VO drop growth.
- Candidate-channel watchdog and heartbeat handling distinguish idle, building, stalled,
  rejected, and accepted states without moving the public update feeds.

Desktop behavior and privacy:

- Flatpak installs without usable DRI nodes select the software renderer instead of failing to
  create a render surface.
- History retention supports Forever, 30 days, 90 days, and one year. Private sessions suppress
  new history, progress, poster, and recent-item writes while preserving existing records,
  matching the Windows retention and privacy contract.

## Install, update, rollback, and uninstall

These artifacts are rejected and must not be published or installed as a stable release. After a
replacement build is accepted, the normal Debian and AppImage installation commands remain:

```bash
sha256sum ok-player_0.1.0-linux-alpha.113_amd64.deb
sudo apt install ./ok-player_0.1.0-linux-alpha.113_amd64.deb
```

```bash
sha256sum OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
chmod +x OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
./OK-Player-0.1.0-linux-alpha.113-x86_64.AppImage
```

Both lanes use the public Linux update feeds only after publication. Roll back by reinstalling an
older `.deb` or running an older AppImage. Uninstall the Debian package with
`sudo apt remove ok-player`; remove an AppImage by deleting the file. Per-user JSON settings and
history remain available unless the user removes them explicitly.

## Known limitations

- The canonical `1120x680` buffered-timeline row places the played and buffered rail centers at
  `627.5px` and the trough center at `632.5px`, a `5px` spread against the `0.1px` tolerance.
- A separate exact-package interaction smoke reproduces the stationary double-click failure: the
  window remains `1120x680` rather than entering `1280x900` fullscreen.
- The independent keyboard-fullscreen smoke passes geometry, visible-video, and chrome-clear
  assertions. It does not override the double-click failure.
- Installed-package, native file/folder chooser, drag/drop, clipboard, desktop portal, compositor
  fullscreen, unavailable always-on-top, keyboard focus, and 4K60 presentation rows are not
  claimed after the deterministic package gate failed.
- Gapless playback, full HDR handling, and A-B clip/GIF export remain explicitly deferred.
